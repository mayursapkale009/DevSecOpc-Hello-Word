From Zero to Production: Deploying a Node.js Application on AWS EC2

A Cybersecurity Student's complete roadmap covering Nginx, PM2, Custom Domains, SSL Certificates, and AWS CI/CD pipelines.

Mayur Sapkale
15 mins read
DevSecOps / Cloud Security

As a cybersecurity student learning cloud and DevOps, I wanted to understand how real-world applications are deployed in production. Instead of only watching tutorials, I decided to build and deploy my own Node.js application on AWS.

This article documents my complete journey, mistakes, errors, solutions, and lessons learned.

Step 1

Launching an AWS EC2 Instance

The journey started by launching an Amazon Linux 2023 EC2 instance. To configure network access, I set up a security group to control incoming traffic:

SSH Port 22 Admin access
HTTP Port 80 Web traffic
HTTPS Port 443 Secure web
Custom TCP Port 3000 Node.js default

Initially, the web application was accessed directly using the public IP on the custom application port:

Initial Access URL 
http://EC2-PUBLIC-IP:3000

Initial verification succeeded! The express landing page rendered successfully.

Step 2

Installing Node.js & Git on Linux

Once the EC2 was running, I connected via SSH and updated the environment, then installed Node.js and Git.

Terminal Setup Commands 
# Connect to EC2
ssh -i key.pem ec2-user@PUBLIC-IP

# Update system package manager (DNF on AL2023)
sudo dnf update -y

# Install Git & Node.js
sudo dnf install git -y
sudo dnf install nodejs -y
Verify Installation 
node -v
npm -v
Step 3

Cloning GitHub Repository

Next, I cloned the project files directly from the GitHub repository into the EC2 instance home directory, installed node packages, and spun up the server:

Cloning & Setup 
# Clone the project code
git clone https://github.com/username/project.git
cd project

# Fetch dependencies and run index.js
npm install
node index.js

The application became responsive and accessible on http://PUBLIC-IP:3000.

Step 4

Configuring PM2 Process Manager

Problem: When the SSH terminal session closed, the active Node process was terminated, crashing the website.
Solution: Install PM2 (Process Manager 2) globally to daemonize the application and run it continuously in the background.
PM2 Core CLI 
# Install PM2 globally
sudo npm install -g pm2

# Start index.js in background under custom label
pm2 start index.js --name index

# Useful Management commands:
pm2 list          # View status of active apps
pm2 logs          # View live logging
pm2 restart index # Restart application
pm2 stop index    # Pause background process

PM2 Key Benefits

  • Auto restart: Instantly recovers app if the process throws an unhandled exception.
  • Process monitoring: Live metrics of memory usage and CPU core allocation.
  • Persistence: Background execution independent of SSH state.
Step 5

Configuring Nginx Reverse Proxy

Problem: Exposing Port 3000 directly to the public is insecure and looks unprofessional (users shouldn't type :3000 in the browser).
Solution: Install Nginx to run on standard HTTP Port 80 and forward requests internally to the Node application on Port 3000.
Install & Start Nginx 
# Install Nginx
sudo dnf install nginx -y

# Enable boot persistence and start service
sudo systemctl enable nginx
sudo systemctl start nginx

Configure the reverse proxy routing rules inside Nginx server blocks:

/etc/nginx/nginx.conf Server Block 
server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
Verify and Reload Nginx 
# Audit config for syntax errors
sudo nginx -t

# Apply settings dynamically without downtime
sudo systemctl reload nginx
User Request Nginx (:80) Node.js (:3000)
Step 6

Learning AWS CodeBuild

To automate building directly from code changes pushed to GitHub, I configured AWS CodeBuild. This compiles, tests, and prepares artifacts for deployment.

Error Encountered: npm ERR! Could not read package.json
Reason: CodeBuild build environment was initialized and ran scripts in the wrong workspace directory.

Key Lessons

  • Always verify the path in CODEBUILD_SRC_DIR env variable.
  • Confirm that your repository repository root structure aligns with build settings.
Step 7

S3 Artifact Region Mismatch

Error Encountered: BucketRegionError
Explanation: The target S3 deployment bucket was created in us-east-1 (N. Virginia), but the active AWS CodeBuild project was executing in ap-south-1 (Mumbai).
DevOps Best Practice

Keep AWS resources (CodeBuild, S3, EC2 instances) inside the same geographical region. This reduces network latency and prevents regional data transfer costs.

Step 8

AWS Inspector Integration Failure

Error Encountered: SubscriptionRequiredException
Explanation: The automated pipeline template attempted to run security scans on the packages using AWS Inspector, but the Inspector service wasn't enabled on my AWS account.

Solution: Edited the pipeline template definition and temporarily bypassed the automated Inspector security stage to avoid blocking the build flow.

Step 9

CodePipeline Empty Test Stage

Error Encountered: npm test - Error: no test specified
Explanation: The default Node.js boilerplate code includes a test script hook. AWS CodePipeline automatically triggers npm test during compilation. Because I hadn't written test files, the script failed and broke the pipeline.

Solution: Configured a placeholder or deleted the test step execution inside package.json scripts:

package.json edit 
"test": "echo test"
Step 10

IAM Trust Relationship Permission Error

Error Encountered: codebuild:StartBuild AccessDeniedException
Explanation: The AWS CodePipeline execution role lacked permissions to call the STS assume API or trigger builds via CodeBuild services.

Solution: Modified IAM roles policies and added explicit trusts for CodeBuild. Most cloud pipeline issues trace back to IAM policy configuration!

Step 11

Simplifying AWS CodeDeploy

Originally, I designed the pipeline using the complete enterprise stack: CodePipeline → CodeDeploy → EC2.

However, configuring CodeDeploy agents and AppSpec YAML structure introduced unnecessary complexity. For learning purposes, a streamlined manual shell workflow was faster and more robust:

Alternative Lightweight Deployment Flow 
# Quick Deploy Command Sequence
git pull
npm install
pm2 restart index

DevOps Lesson: Don't over-engineer. Sometimes, simple and manageable setups are better than fully automated but complex pipelines, especially during early project phases.

Step 12

Custom Domain Integration

To give the application a professional look, I purchased a domain and mapped a specific subdomain pointing to the server:

DNS Record Details

  • Target Subdomain: mayur.cybershieldd.online
  • Record Type: A Record (Address Mapping)
  • Value: 3.6.58.203 (EC2 Elastic Public IP)

After mapping, domain behaviors were inconsistent, sometimes failing to resolve.

Step 13

DNS Propagation & Multiple A-Records

I investigated the DNS using terminal lookup tools like dig and nslookup:

DNS Diagnostic Lookup 
dig mayur.cybershieldd.online
Root Cause: The lookup revealed that multiple legacy A-Records were still attached to the root domain mapping: 
15.197.142.173
3.33.152.147
3.6.58.203
Fix: Cleared all duplicate registrar records, keeping only the single correct Elastic IP pointer to resolve redirect loops and SSL failures.
Step 14

HTTP working but HTTPS failing

At this stage, the domain resolved correctly over insecure HTTP (http://mayur.cybershieldd.online), but request connections over secure HTTPS port 443 were rejected.

Verify Local SSL Certificates 
sudo certbot certificates

Output: No certificates found. I needed to request and install a public TLS/SSL certificate.

Step 15

Requesting & Installing SSL Certificate

I installed **Certbot** and its Nginx automation plugin to request a free certificate from the Let's Encrypt authority:

Certbot Installation & Request 
# Install packages via DNF
sudo dnf install certbot python3-certbot-nginx -y

# Request certificate and let Certbot auto-configure Nginx
sudo certbot --nginx -d mayur.cybershieldd.online

Verification curl request shows the certificate validation is correct:

HTTPS Header Check 
curl -I https://mayur.cybershieldd.online

Connection Success! Let's Encrypt CA certificate is active, securing all application traffic.

Conclusion & Takeaways

What started as a simple Express script evolved into a production-ready application environment on AWS. Cloud engineering isn't about memorizing dashboard buttons; it's about understanding how systems connect and how to troubleshoot infrastructure when configurations fail.

01

Config is Key

Most AWS issues are small configuration or networking rule mismatches rather than bad code.

02

DNS First

Always verify DNS mappings using command utilities (dig, nslookup) instead of trusting provider dashboard UIs.

03

IAM Security

AWS pipelines require detailed trust configurations. IAM permissions break a majority of builds.

Troubleshooting Lab

Quick lookup guide for errors encountered during this project build. Search below to filter issues by services or terms.

Error 1

SSH Key Permissions Too Open

Console Warning:

@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@         WARNING: UNPROTECTED PRIVATE KEY FILE!          @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
Permissions for 'MayurKey.pem' are too open.
It is required that your private key files are NOT accessible by others.

Actionable Solution:

In UNIX-like file systems, SSH private keys must have strict user-only read permissions. Secure the file permissions using chmod:

CLI Fix 
chmod 400 MayurKey.pem
Error 2

Website Refused Connection

Console Warning:

ERR_CONNECTION_REFUSED
Failed to load resource: net::ERR_CONNECTION_REFUSED

Actionable Solution:

Ensure that your Express application is actively running and listening on the specified port. Also confirm the EC2 Security Groups allow incoming connections on that port.

Sample Express Server Config 
const express = require('express');
const app = express();

app.get('/', (req,res) => {
    res.send("Active Hello World");
});

app.listen(3000, () => {
    console.log("Server Running on Port 3000");
});
Error 3

PM2 Process Already Exists

Console Warning:

[PM2][ERROR] Script already launched, use 'pm2 restart index' or 'pm2 reload index'

Actionable Solution:

Instead of executing pm2 start repeatedly, view running processes using pm2 list and restart the existing entry.

CLI Fix 
# Show active processes
pm2 list

# Restart the application safely
pm2 restart index
Error 4

Package.json Missing

Console Warning:

npm ERR! code ENOENT
npm ERR! syscall open
npm ERR! path /home/ec2-user/project/package.json
npm ERR! errno -2
npm ERR! enoent ENOENT: no such file or directory, open 'package.json'

Actionable Solution:

This happens when running npm command inside directories that do not contain project packages. Initialize a new project structure or navigate to the repository directory.

CLI Fix 
# Generate package.json in the current working directory
npm init -y
Error 5

S3 Bucket Region Mismatch

Console Warning:

BucketRegionError:
The bucket you are attempting to access is in region us-east-1,
but the client is configured to connect to region ap-south-1.

Actionable Solution:

Create S3 deployment storage buckets in the exact region where your CodeBuild service project operates.

AWS CLI Bucket Creation 
aws s3api create-bucket \
    --bucket my-devops-bucket \
    --region ap-south-1 \
    --create-bucket-configuration LocationConstraint=ap-south-1
Error 6

CodePipeline Test Failure

Console Warning:

npm ERR! Lifecycle script `test` failed with error code 1.
npm ERR! Failed at the test script 'echo "Error: no test specified" && exit 1'.

Actionable Solution:

Boilerplate code exit scripts block automated builders. Replace the default exit placeholder inside the package.json file.

package.json scripts block 
"scripts": {
  "test": "echo \"Test Skipped\""
}
Error 7

CodeBuild STS AssumeRole Denied

Console Warning:

CodeBuild is not authorized to perform sts:AssumeRole for role arn:aws:iam::123456789:role/PipelineRole

Actionable Solution:

Include the codebuild.amazonaws.com principal entity inside the trust relationship document schema linked to the AWS role.

Trust Relationship Policy Document 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "codebuild.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
Error 8

GitHub Webhook Access Denied

Console Warning:

Webhook creation failed. Access Denied: Developer OAuth integration lacks scope 'admin:repo_hook'

Actionable Solution:

In the AWS Console, re-authorize your GitHub repository connection. Use AWS CodeConnections and grant full administration scopes for webhooks so AWS can trigger pipeline builds automatically on repository pushes.

Error 9

Inspector Scan Failed

Console Warning:

SubscriptionRequiredException:
Account is not subscribed to Amazon Inspector.
Please activate the service in the AWS dashboard.

Actionable Solution:

AWS Cloud template templates often invoke Amazon Inspector by default. If your account is on the Free Tier and doesn't use Inspector, edit the pipeline JSON templates or CloudFormation build steps to remove or comment out the security audit stage.

Error 10

CodeDeploy Agent Missing

Console Warning:

InstanceAgent::Plugins::CodeDeployPlugin::CommandPoller:
No agent configuration or service 'codedeploy-agent' is inactive.

Actionable Solution:

Login to the target EC2 node via SSH. Install Ruby and download the regional AWS CodeDeploy package script wrapper to run agent services:

EC2 SSH Installer Commands 
# Install dependencies
sudo dnf install ruby wget -y

# Download install wrapper from Amazon S3 (Mumbai Region example)
wget https://aws-codedeploy-ap-south-1.s3.ap-south-1.amazonaws.com/latest/install
chmod +x install

# Run installer script
sudo ./install auto

# Confirm service is up
sudo systemctl status codedeploy-agent
